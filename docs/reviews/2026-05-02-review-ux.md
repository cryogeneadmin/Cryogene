# UX & Visual Design Review — Cryogene Storefront, 2026-05-02

Reviewer: UX Architect subagent · Brand: editorial/boutique pharmacopoeia (NOT Shopify-template). Hard prohibitions: no drop shadows, no gradients, no rounded buttons, no lifestyle imagery, no testimonials, no countdown timers, no email popups.

## CRITICAL

### C1. Brand fonts don't match spec
`app/layout.tsx:2` imports `Playfair_Display` + `Inter`; `app/globals.css:35-36` declares `--font-serif` as `var(--font-playfair)`. Spec says **Cormorant Garamond** + **DM Sans**. Comment in `globals.css:34-35` even mentions Cormorant in fallback list — someone knew, shipped Playfair anyway.

Playfair is high-contrast didone (Bodoni cousin); Cormorant is transitional with softer pharmacopoeia register. They read as different brands. Fix: swap to `Cormorant_Garamond` + `DM_Sans`. Site-wide impact, 10-line change.

### C2. Decorative gradients + drop shadows on molecular thumbnail
`ProductCard.tsx:94` and `ProductDetail.tsx:115`: `bg-gradient-to-br from-[#F0F4FA] via-[#E6ECF5] to-[#CAD4E4] … rounded-sm shadow-[0_4px_10px_-2px_rgba(13,27,62,0.25)]`. Detail variant adds `drop-shadow-md` to the molecule image (`ProductDetail.tsx:121`).

Three brand violations (gradients, drop shadows, rounded) in one element repeated 47+ times. Reads as Shopify-template polish. Fix: flat off-white panel with hairline border, no shadow.

### C3. Cookie consent banner has top drop shadow
`CookieConsent.tsx:24`: `shadow-[0_-4px_12px_rgba(13,27,62,0.06)]`. Direct violation. Remove; rely on `border-t border-[#DDE1E7]`.

### C4. Mobile filter drawer uses `shadow-xl`
`ListingToolbar.tsx:89`. Brand violation, unnecessary because dark overlay separates already.

### C5. `StorageHandlingPanel` dynamic Tailwind class won't compile
`StorageHandlingPanel.tsx:45`: `md:grid-cols-${specs.length}`. Tailwind v4 statically extracts class names; runtime-interpolated never generated. Supplies (2 specs) and mixers (4 specs) silently fall back. Functional bug. Fix: `cn()` lookup — `specs.length === 2 ? "md:grid-cols-2" : ...`.

### C6. AdminSidebar broken active-state ternary
`AdminSidebar.tsx:27-33`. Expression `pathname === link.href ? : ... : ...` collapses ternary to boolean for dashboard link. Dashboard always renders without active styling; other pages get className from boolean evaluation. Functional bug. Fix: `const isActive = link.href === "/admin" ? pathname === "/admin" : pathname.startsWith(link.href);` + `aria-current="page"`.

## HIGH

### H1. Design tokens defined but ignored — 307 hard-coded hex literals across 42 files
`@theme` declares `--color-navy`, `--color-border`, `--color-offwhite` etc. Tokens used via Tailwind utilities only in `components/ui/button.tsx` and `badge.tsx`. Storefront bypasses: `bg-[#0D1B3E]`, `border-[#DDE1E7]`, `text-[#6B7280]` repeated dozens of times. Palette tweak requires touching 42 files. Codemod-friendly 1:1 map.

### H2. Mono font misuse
Spec: mono only for CAS / formula / SKU / order#. Violations:
- `ProductDetail.tsx:126` — "Tested by HPLC" `font-mono uppercase`. Method label, not data.
- `ListingToolbar.tsx:51` — active filter count badge `font-mono`. UI counter.
- `CompoundStatsBar.tsx:39` — purity `≥99%` flagged `mono: true`. Measurement, not structural id.
- `BlendedProductComposition.tsx:15` — composition amount `5mg` mono. Borderline.

Strip from purity, testing method, filter count, composition amount. Keep on CAS, formula, MW, SKU, order#, basket sub-line.

### H3. No checkout step indicator
`checkout/delivery/page.tsx:12` — *Step 1 of 2* eyebrow caption. Confirmation has no progress at all. No visual progress bar, no breadcrumb-like step list. Add hairline three-dot indicator with serif labels.

### H4. ProductFAQ all-expanded
Acceptable for accessibility but creates long scroll on products with 5–8 FAQ. Use Base UI Collapsible (already in `components/ui/collapsible.tsx`); first FAQ open, rest collapsed. JSON-LD remains source of SEO truth.

### H5. Product detail no image gallery
`ProductDetail.tsx:78-88` always renders `primaryImage` only. `Product.images` array unused. Lost editorial opportunity. Add vertical thumbnail rail (left of hero on desktop; below on mobile), 3–4 thumbs max, click-to-swap, no carousel arrows.

### H6. Quantity buttons fail 44px tap-target
`BasketItem.tsx:33,42` — `w-7 h-7` = 28×28. Bump to `w-11 h-11` or `min-h-[44px]`.

### H7. Listing breadcrumb verified working — listed under strengths.

### H8. Empty basket page omits "Sign in to recover" cue
`app/(public)/basket/page.tsx:11-34`. Add tertiary link below CTA.

## MEDIUM

- Compliance banner fixed `top-0` + `pt-9`. On viewports needing wrap, banner takes 2-3 lines but `pt-9` fixed. Tighten copy or measure dynamically.
- ProductImageShell skeleton uses sweeping gradient shimmer (`105deg ... rgba(192,200,216,0.35)`). Brand violation. Replace with fade-in.
- AccountLayout sign-out renders identical to nav links. Render as tier-2 action.
- Admin status badges use `rounded` + Tailwind defaults. Either explicitly off-brand or use compliance amber + success green tokens.
- Anchor nav `sticky top-28` collides with sticky navbar shrink — small white strip.
- "Recently added" section copy weak vs brand register.
- ConfirmationContent lacks order-status timeline — confirmation is emotional moment.
- Filter drawer no per-checkbox count for sizes/methods.
- No `loading.tsx` skeleton on listing pages.
- Basket drawer empty state mostly fallback path — fine.

## LOW

- Footer credibility row — four hand-drawn SVGs not visually unified.
- Hero 50/50 split — try `md:grid-cols-[7fr_5fr]` for editorial asymmetry.
- CompoundStatsBar mobile-wrap border math leaves orphan cell mismatched.
- BasketIconButton count pill rounded-full — codify exemption or square + hairline.
- Sign-in/sign-up CTAs underline-only — start navy.

## Strengths to preserve

- **`label-editorial` typography utility** (`globals.css:93-99`) used consistently across nav, breadcrumbs, hero eyebrows, section labels — backbone of boutique register.
- **Hairline border + flat panel discipline** on cards, stat bars, storage panel, review summary — exactly the brand spec.
- **Compliance amber palette** consistent across age gate, research disclaimer, stub-payment notice, placeholder banner, confirmation reminder. One amber, one purpose.
- **Breadcrumb implementation** on detail pages with custom chevron, label-editorial styling, current in navy. Quietly excellent.
- **CompoundStatsBar** is the strongest single piece of editorial design — five fields in hairline grid with serif labels and mono values. Aspire here, not the molecule panel.
- **Cookie banner equal-weight buttons** (ICO).
- **Filter sidebar with `accent-[#0D1B3E]`** — native checkboxes inheriting brand color. Restrained.
- **Server actions for cookie/age gate consent** — cookie-based, no client JS for decision.
- **Sticky checkout summary** — keeps order context visible.
- **Molecular structure surfacing on-brand pharmacopoeia** — just needs flat treatment with grid lines, deep navy ink (no gradients/shadows).
